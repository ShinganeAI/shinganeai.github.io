Part III: History of Machines That Think

Building a shared mental model of computation.

Prelude: The Nature of Thought

"The modern framework for computation emerged from a thought experiment exploring the nature of thought itself."
Alan Turing, "Computing Machinery and Intelligence" (1950)

This milestone addresses the question **"Can Machines Think?"** and defines the separation of the *process* from the *processor*.

Origins: Discrete vs. Continuous

The Abacus (2700 BCE)

Turned arithmetic into a physical process using columns for units and tens. Focused on the Discrete: "How many?"

[Abacus Illustration]

Antikythera (2nd c. BCE)

A bronze gear system modeling celestial motion. Focused on the Continuous: "Where will it be?"

[Antikythera Gears Illustration]

Babbage & Lovelace: The DNA of Software

The Engines

  • Difference Engine: Designed to evaluate polynomials via finite differences.
  • Analytical Engine: The first design for a general-purpose programmable computer.
  • Architecture: Featured a **Store** (memory) and a **Mill** (processor).

Ada Lovelace (1843)

  • Created the first algorithm (calculating Bernoulli numbers).
  • Recognized machines could manipulate *any* symbols, not just numbers.
  • Laid the foundation for the Universal Computer.

The Theoretical Crucible

Hilbert’s Dream

David Hilbert sought to place all math on a rigorous axiomatic basis:

  • Consistency: No contradictions.
  • Completeness: Every true statement provable.
  • Decidability: An algorithm to determine truth.

Gödel’s Revelation (1931)

Kurt Gödel published the **First Incompleteness Theorem**, proving that any system contains true statements that cannot be proven within it.

His Second Theorem proved a system cannot prove its own consistency.

The Universal Turing Machine

The Decision Problem

To address the *Entscheidungsproblem*, Turing defined a formal model: an infinite tape, a read/write head, and a control unit.

The Halting Problem: Determining if a machine halts is undecidable. This proved the Decision Problem itself is undecidable.

Church–Turing Thesis

If a problem can be solved by a human following finite steps, a Turing machine can solve it. Program descriptions are treated as data.

Harvard vs. Von Neumann

Harvard Architecture

Harvard Mark I (1944): Instructions stored externally on paper tape; data in storage counters. Instructions and data remain physically separate.

Von Neumann (1945)

Stored-Program Concept: Instructions reside in the same memory as data. Instructions become just another kind of data—the Universal Machine in hardware.

This unification enabled compilers, operating systems, and the flexibility of modern software.

Miniaturization & The Shift to Software

[CPU Illustration]

The Intel 8086/88

Miniaturized stored-program power into chips. The IBM PC ecosystem turned general-purpose computing into a global desktop standard.

The Software Discipline: Standardized architecture shifted innovation from hardware to algorithms and data structures.

Shingane represents the next step: using software intelligence to reclaim hardware design.
